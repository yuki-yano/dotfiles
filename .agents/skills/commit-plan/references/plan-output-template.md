# 計画出力テンプレート

## コミット計画の形式

```text
コミット計画

[1] <type>: <サマリ>
- <file1>
- <file2> [部分] Lxx-Lyy

実行コマンド:
git add <file...>
git commit

コミットメッセージ:
<type>: <サマリ>

- 箇条書き1
- 箇条書き2
```

## 計画報告で必ず含める項目

1. 自然言語指示の解釈
2. 計画した総コミット数
3. 各コミットの内容
4. 意図別内訳
5. 部分ステージングが必要なファイル数

## 実行確認の固定フォーマット

```text
この計画を実行しますか？
- y: Phase 2 を開始
- e: 計画を修正
- n: 計画のみで終了
```

`--auto` が指定されていても、この確認は実施する。
