# 計画出力テンプレート

## コミット計画の形式

```text
コミット計画

解釈:
- 対象範囲: <全体 / pathspec>
- 実行意図: <計画のみ / 承認後に実行>

[1] <type>: <サマリ>
- <file1>
- <file2> [部分] Lxx-Lyy
- staged: <あり / なし>
- untracked: <あり / なし>

実行コマンド:
git add -- <file...>
git commit

コミットメッセージ:
<type>: <サマリ>

- 箇条書き1
- 箇条書き2
```

## 計画報告で必ず含める項目

1. 自然言語指示の解釈
2. 計画した総コミット数
3. 各コミットの内容
4. 意図別内訳
5. 部分ステージングが必要なファイル数
6. staged / unstaged / untracked の扱い
7. 対象外差分と停止条件の有無
8. 直近 commit log を踏まえたコミットメッセージ方針

## 注意事項の形式

対象外差分や判断不能な差分がある場合は、コミット計画とは別に必ず表示する。

```text
確認が必要な差分
- <path>: <理由>
```

確認が必要な差分がある場合は、Phase 2 に進めない。

## 実行確認の固定フォーマット

```text
この計画を実行しますか？
- y: Phase 2 を開始し、計画した全コミットを順次実行
- e: 計画を修正
- n: 計画のみで終了
```

計画のみの依頼では、この確認フォーマットを出さず、計画提示で終了する。
