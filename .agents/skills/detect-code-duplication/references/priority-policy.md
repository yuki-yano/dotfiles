# Priority Policy

`similarity-ts` の出力を次の基準で分類する。

## 基本閾値

- 標準スキャン: `threshold=0.60`
- 広域スキャン（ノイズ多）: `threshold=0.40`

## 優先度判定

- 高優先度
  - 類似度 `>= 0.80`
  - かつ 3 箇所以上に波及、またはクリティカル経路（本番ロジック、主要 API、共通基盤）に存在
- 中優先度
  - 類似度 `0.70 - 0.79`
  - 2 箇所以上に波及
- 低優先度
  - 類似度 `0.60 - 0.69`
  - 影響が局所的

## 除外候補の扱い

次は「削除対象」ではなく「維持理由付き記録」に回す。

- 生成コード
- テストフィクスチャ
- 互換維持のための明示的重複
- 型のミラー実装（段階移行中など）

## 記録ルール

計画書には各候補で次を記載する。

- 類似度
- 影響ファイル
- 推奨アクション（共通化/維持）
- 根拠（なぜその判断か）
