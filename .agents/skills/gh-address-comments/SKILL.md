# PR コメント対応

現在のブランチに紐づくオープン PR を見つけ、そのコメントに `gh` CLI で対応するためのガイドです。`gh` コマンドはすべて、ネットワーク権限を有効にして実行してください。

事前条件: `gh` が認証済みであることを確認します（たとえば一度 `gh auth login` を実行しておく）。そのうえで、`gh` コマンドが成功するよう、権限を引き上げた状態で `gh auth status` を実行してください（`workflow` と `repo` スコープを含む）。サンドボックスが `gh auth status` を妨げる場合は、`sandbox_permissions=require_escalated` を付けて再実行します。

## 1) 対応が必要なコメントを確認する
- `scripts/fetch_comments.py` を実行します。PR 上のすべてのコメントとレビュー スレッドが出力されます。

## 2) ユーザーに確認する
- すべてのレビュー スレッドとコメントに番号を振り、それぞれに対応するために必要な修正内容を短く要約します。
- どの番号のコメントに対応するかをユーザーに確認します。

## 3) ユーザーが対象コメントを選んだら
- 選ばれたコメントに対応する修正を適用します。

補足:
- 途中で `gh` が認証エラーやレート制限に当たった場合は、`gh auth login` で再認証するよう促してから再試行します。
